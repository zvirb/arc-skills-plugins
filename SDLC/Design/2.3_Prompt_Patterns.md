# 2.3 Prompt Design Patterns

## Objective
To implement specialized reasoning structures that maximize the performance of legacy GPU models (Qwen, DeepSeek).

## Pattern Catalog

### 1. The Blindfold Prompt (Execution Nodes)
- **Constraint**: Strip out all conversational ability.
- **Instruction**: "Your sole purpose is to map input variables to the following JSON schema. Do not explain your actions. Do not provide conversational feedback."
- **Benefit**: Reduces token waste and VRAM usage on worker nodes.

### 2. The Chain of Thought (CoT) Wrapper (Orchestrator)
- **Usage**: Use on the Qwen 3.6 Orchestrator (Pascal).
- **Instruction**: "Before calling a tool, explicitly state your reasoning and a step-by-step plan in the `reasoning` field."
- **Benefit**: Increases reliability of complex routing decisions.

### 3. The Critic/Refiner Loop
- **Architecture**: A two-node sequence where Node A (Draft) generates output and Node B (Critic) validates it against a checklist.
- **Benefit**: Implements Kaizen by identifying and correcting subtle hallucinations before they propagate.

### 4. The Andon Prompt (Error Correction)
- **Trigger**: Triggered by a tool failure.
- **Instruction**: "The tool returned the following error: <error>. Analyze this failure and generate a corrected tool call. Max retries remaining: <n>."
