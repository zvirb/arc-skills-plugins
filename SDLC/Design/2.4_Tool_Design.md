# 2.4 Tool & API Design

## Objective
To design deterministic, hardware-optimized interfaces for agentic actuators.

## Design Rules for Tools

### 1. Static Deterministic Binding (OpenClaw Standard)
- **Standard**: For .lobster workflows, bypass probabilistic JIT discovery. 
- **Rule**: Explicitly declare required tools in the `tools.alsoAllow` array in `openclaw.json` or the plugin manifest.
- **Benefit**: Eliminates latency and semantic drift by forcing immediate tool schema injection at initialization.

### 2. Lazy Evaluation & Initialization
- **Standard**: Heavy capability factories (e.g., PDF extraction, Browser binaries) must be removed from the critical reply path.
- **Rule**: Only load and initialize tool factories if the explicitly authenticated provider is available. Delay binary loading until the exact microsecond of invocation.

### 3. Idempotency (Standardized Work)
- Running the same tool with the same arguments multiple times should produce the same state change.
- Example: Use `gog tasks create --if-not-exists` patterns to prevent duplicate side effects.

### 4. Non-Destructive Verification
- **CRITICAL RULE**: Verification tools (e.g., `list`, `get`) must ONLY observe state. They are strictly forbidden from modifying or completing tasks.

### 5. CLI Safety & Serialization
- Include `--no-input` and `--force` flags.
- **Serialization**: Tools must support the **PluginArtifact** metadata schema to avoid 10KB JSON-RPC string limits.

## Hardware Resource Mapping

| Role | GPU Node | Recommended Model | Primary Function |
| :--- | :--- | :--- | :--- |
| **Orchestrator** | Pascal (24GB) | Qwen 3.6 35B | Workflow logic, state maintenance. |
| **Specialist** | Maxwell (12GB) | Qwen 2.5 Coder 7B | Atomic tool calls, code generation. |
| **Vision/OCR** | Maxwell (12GB) | DeepSeek-OCR 2 | Visual extraction, DOM parsing. |
