# SDLC Domain 7: Evolution

## Purpose
The Evolution domain focuses on "Kaizen" (Continuous Improvement)—analyzing performance data to refine prompts, optimize model selection, and implement recursive self-improvement loops. This phase ensures the system evolves to meet changing requirements and higher quality standards.

## Documents
- **[7.1 Performance Benchmarking](./7.1_Benchmarking.md)**: Measuring behavioral success and resource efficiency.
- **[7.2 Data-Driven Kaizen](./7.2_Kaizen.md)**: Using telemetry to identify and fix systemic weaknesses.
- **[7.3 Recursive Optimization](./7.3_Recursive_Optimization.md)**: Implementing self-referential improvement loops.
- **[7.4 Refactoring & Technical Debt](./7.4_Refactoring.md)**: Managing the lifecycle of aging skills and plugins.

## Workflows
- `evo-bench`: Run the "Agent-as-a-Judge" evaluation suite.
- `evo-optimize`: Automated prompt refinement based on failure logs.
- `evo-refactor`: Guided decomposition of monolithic "waste" into atomic nodes.

## Jidoka Gate
The Evolution phase identifies the next "Standardized Work" baseline. It is considered successful when a **Performance Lift** (e.g., +10% accuracy or -20% VRAM) is demonstrated and documented in the project's historical logs.
