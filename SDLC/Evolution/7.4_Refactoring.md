# 7.4 Refactoring & Technical Debt

## Objective
To manage the lifecycle of the repository and eliminate monolithic "waste" that accumulates over time.

## Refactoring Triggers
- **Node Bloat**: A skill or plugin exceeding 300 lines.
- **Cognitive Complexity**: An agent failing to complete a task after 5 turns.
- **Resource Exhaustion**: A node requiring a context window larger than the hardware-allocated cap.

## The "Purge" Protocol (Standardized Work)
1.  **Identify Monoliths**: Search for skills that combine fetching, logic, and actuation.
2.  **Decompose**: Break the monolith into at least 3 atomic nodes (Fetcher, Transformer, Actuator).
3.  **Validate**: Ensure the new atomic chain has parity with the original monolithic function.
4.  **Archive**: Move the legacy monolith to the `Archive/` directory.

## Jidoka Check
- [ ] Is the root directory clean of temporary or legacy files?
- [ ] Are all active skills and plugins following the Atomic Node pattern?
