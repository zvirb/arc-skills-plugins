# 7.2 Data-Driven Kaizen

## Objective
To implement continuous, incremental improvements based on evidence from operational logs.

## The Kaizen Loop

### 1. Identify Muda (Waste)
Analyze logs for:
- Redundant tool calls.
- Excessive conversational filler.
- Inefficient context loading (Cache Thrashing).

### 2. Perfect the Atomic Step (善)
- Refine the **Blindfold Prompt** for a specific node to reduce token usage.
- Update a tool's schema to be more explicit, reducing extraction errors.

### 3. Standardize the Improvement
- Propagate the optimized prompt or schema across all similar nodes in the system.
- Update the **SDLC Design patterns (2.3)** to reflect the new best practice.

## Case Study: VRAM Optimization
If the Maxwell node consistently hits OOM during OCR tasks, Kaizen dictates:
- Reducing the visual token compression ratio in `DeepSeek-OCR`.
- Hard-capping the concurrent worker count to 1.
