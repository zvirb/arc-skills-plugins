# 6.1 Infrastructure Monitoring & Gateway Health

## Objective
To ensure the physical hardware and Node.js runtime are operating within safe parameters for sustained autonomous execution.

## Monitoring Metrics (Pascal/Maxwell Node)

| Metric | Target | Alert Threshold | Remediation |
| :--- | :--- | :--- | :--- |
| **VRAM Utilization** | < 80% | > 95% | Trigger Cache Flush; restart model instance. |
| **GPU Temperature** | < 75°C | > 85°C | Suspend background cron tasks for 5 mins. |
| **PCIe Bandwidth** | Constant | Spikes > 80% | Disable Tensor Parallelism; check CNI routing. |
| **Event Loop Lag** | < 50ms | > 300ms | Enable `heavy_task_offload_enabled`; check PDF/Browser tools. |

## Runtime Stability Standard (Jidoka)

### 1. Heavy Task Offload
- **Mandatory**: Set `heavy_task_offload_enabled: true` in `openclaw.json`.
- **Reason**: Shifts CPU-intensive skill management and binary instantiations to **Worker Threads**, preventing the main event loop from stalling.

### 2. Event Loop Monitor
- **Standard**: Configure `event_loop_monitor_threshold_ms: 300`.
- **Action**: If a stall is detected, the gateway must log a `STALL_WARNING` and identify the unsettled top-level await causing the delay.

### 3. Session Lock Watchdog
- **Standard**: Monitor for `SessionWriteLockTimeoutError`.
- **Rule**: If a lock lease expires without a clean exit, the Watchdog must purge the stale `.lock` file and report the zombie process ID.

## Jidoka Rule: Automated Throttling
If VRAM utilization on the Maxwell node exceeds 95% or PCIe bandwidth saturates:
1.  The system must automatically queue new worker tasks in the lane queue.
2.  The Orchestrator must be restricted to a **4,096 token** transient context window.
3.  All non-essential vision tasks (OCR) must be deferred.
