# 6.4 Maintenance & Drifts

## Objective
To maintain system integrity and prevent "Behavioral Drift" or "Configuration Corruption" over time.

## Drift Detection

### 1. Configuration & Redaction (CRITICAL)
- **Redaction Lockout**: If `openclaw.json` is modified by an agent, it will overwrite API keys with `***`.
- **Maintenance**: Periodically verify `openclaw.json` against the `Secrets/` backup. Never allow automated writes to the config.

### 2. Dependency & Symlink Stability
- **Symlink Trap**: OpenClaw severs symlinks when writing `openclaw.json`.
- **Maintenance**: Ensure GitOps symlinks are intact. Run `openclaw doctor --fix` ONLY to repair the registry, then manually restore custom overrides.

### 3. Model & KV Cache Stability
- **Maintenance**: Weekly monitor for `SessionWriteLockTimeoutError`. If frequency increases, defragment VRAM by flushing the `Q8_0` KV cache and restarting the Maxwell worker node.

### 4. Logic & Prompt Drift
- **Maintenance**: Re-run the **Deterministic Evaluation (SDLC 4.1)** monthly to detect logic shifts caused by model backend updates (e.g., Ollama v0.1.x to v0.2.x).

## Maintenance Checklist
- [ ] Restore `openclaw.json` from Git source of truth if `***` redactions appear.
- [ ] Verify `heavy_task_offload_enabled: true` persists after gateway restarts.
- [ ] Run `openclaw skills check --all` weekly to verify tool eligibility.
- [ ] Rotate API keys in `Secrets/` every 90 days.
- [ ] Perform a "House Tidy" of `Logs/` and `Memory/` every 30 days.
