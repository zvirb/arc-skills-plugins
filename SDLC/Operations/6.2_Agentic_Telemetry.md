# 6.2 Agentic Telemetry

## Objective
To peer into the "Black Box" of agentic reasoning to identify failure points and logic errors.

## Telemetry Pillars

### 1. Decision Tracing
- Log every "Reason -> Plan -> Act" cycle.
- Capture the "Internal Thought" tokens that precede a tool call.

### 2. Tool I/O Auditing
- Log the raw input payload and the raw output response for every tool execution.
- Capture any "Invisible Failures" (e.g., tool exits 0 but returns no data).

### 3. Token Economy
- Track total prompt and completion tokens per session.
- Identify "Context Bloat" where prompts become unnecessarily long over time.

## Trace Analysis Workflow
1.  **Extract Trace**: Use `openclaw logs --session <id>`.
2.  **Analyze Logic**: Map the decision steps against the **Design Specification (SDLC 2.1)**.
3.  **Identify Drift**: Flag any deviations from the intended behavior pattern.
