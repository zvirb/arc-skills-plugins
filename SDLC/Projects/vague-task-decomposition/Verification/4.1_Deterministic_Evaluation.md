# 4.1 Deterministic Evaluation: Vague Task Decomposition

## Success Criteria

| Node ID | Success Metric | Verification Method |
| :--- | :--- | :--- |
| `NODE_EXTRACT` | Valid JSON Array. | `JSON.parse()` check in verification script. |
| `NODE_VALIDATE` | Correct number of items (>1). | Length check. |
| `NODE_APPROVAL` | "Approved" token received. | String matching. |
| `NODE_TASK_ADD` | `gog` status `SUCCESS`. | Exit code 0 + "Created task" in stdout. |
| **End-to-End** | Tasks visible in Google Tasks. | `gog tasks find` or Browser verification. |

## Verification Plan

### 1. Atomic Node Test (`NODE_EXTRACT`)
- **Input**: "Organize my birthday party"
- **Expected**: `["Find a venue", "Invite friends", "Buy cake"]` (or similar).
- **Tool**: `npx openclaw chat --agent operator`

### 2. Independent State Confirmation
- **Tool**: `gog tasks list @default`
- **Action**: Verify that the exact subtasks returned by `NODE_EXTRACT` are present in the output.
