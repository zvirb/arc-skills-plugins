# 1.3 Decomposition Planning: Morning Briefing Refactor

## Master Atomic Node List

### 1. Data Retrieval Nodes (Maxwell/Direct Tool)
- **`NODE_CAL_FETCH`**: Refactor `google-calendar-find-event` to call `gog calendar list --time-min ... --time-max ...` directly.
- **`NODE_TASK_FETCH`**: Refactor `google-tasks-find-tasks` to call `gog tasks list @default --status needsAction` directly.
- **`NODE_WEATHER_FETCH`**: Call `weather-get-forecast` directly.

### 2. Planning & Strategy Nodes (Pascal/Orchestrator)
- **`NODE_TIME_BLOCK`**: Specialized prompt to map `NODE_CAL_FETCH` and `NODE_TASK_FETCH` outputs into suggested calendar blocks.
- **`NODE_SUMMARIZE`**: Generate the WhatsApp-friendly digest from the planned blocks.

### 3. Actuator Nodes (Direct Tool)
- **`NODE_WA_NOTIFY`**: Send the digest via WhatsApp.
- **`NODE_CAL_COMMIT`**: Refactor `google-calendar-create-event` to call `gog calendar create ...` directly for approved blocks.

## Workflow Topology (Lobster)
1. **Step 1: Context Collection**: Run `NODE_CAL_FETCH`, `NODE_TASK_FETCH`, and `NODE_WEATHER_FETCH` in parallel.
2. **Step 2: Scheduling**: Pass results to `NODE_TIME_BLOCK`.
3. **Step 3: Notification**: Run `NODE_SUMMARIZE` and then `NODE_WA_NOTIFY`.
4. **Step 4: Approval**: `wait_for_approval` via WhatsApp.
5. **Step 5: Execution**: Run `NODE_CAL_COMMIT` for approved items.
