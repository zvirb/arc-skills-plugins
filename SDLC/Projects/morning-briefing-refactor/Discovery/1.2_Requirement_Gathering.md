# 1.2 Requirement Gathering: Morning Briefing Refactor

## Functional Requirements
1. **Multi-Source Aggregation**: Must pull from Google Calendar (@primary) and Google Tasks (@default).
2. **Environmental Awareness**: Must fetch current and forecast weather for the user's location.
3. **Intelligent Time-Blocking**: Must propose time blocks for tasks based on available slots in the calendar.
4. **Conflict Resolution**: Must identify and report overlapping events.
5. **Interactive Approval**: Must send a draft schedule to the user via WhatsApp and wait for approval/modification.
6. **Persistence**: Must create actual calendar events for the approved task blocks.

## Technical Requirements
1. **Direct Tool Calling**: No shell script wrappers. Call `gog` and `weather` directly via `exec`.
2. **Asynchronous Processing**: Use `step` separation to prevent blocking.
3. **Memory Management**: Use `INT4` models for extraction nodes to stay within Maxwell VRAM (12GB).
4. **Authentication**: Use `oauth2` for Google services (managed by `gog`).
5. **Error Handling**: Implement deterministic retries for API failures (503/429).
