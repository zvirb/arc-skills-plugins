# 1.4 Feasibility & Risk Analysis: Morning Briefing Refactor

## VRAM Assessment (Maxwell/12GB)
- **Model**: Qwen 7B (INT4).
- **Footprint**: ~5.5GB (VRAM).
- **Context**: 4k limit (~1.2GB KV Cache).
- **Result**: **FEASIBLE**. Leaves 5.3GB for vision/micro-agents.

## API Governance
- **Google Tasks/Calendar**: Rate limits are high (10k/day). **FEASIBLE**.
- **Weather**: BOM/OpenWeather free tier (60 calls/min). **FEASIBLE**.
- **WhatsApp**: Local gateway rate limits depend on signal/connectivity.

## Identified Risks
- **Data Drift**: Calendar changes between generation and approval (Step 2 and Step 5).
  - *Mitigation*: Re-fetch calendar state in `NODE_CAL_COMMIT` before final write.
- **Authentication**: `invalid_grant` during background cron run.
  - *Mitigation*: Implement `openclaw-doctor` hook to check GOG token health every 24h.
- **LLM Hallucination**: AI proposing impossible blocks (e.g., 25th hour of the day).
  - *Mitigation*: Enforce strict ISO-8601 regex validation in the planning node output.
