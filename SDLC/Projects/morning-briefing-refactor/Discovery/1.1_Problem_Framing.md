# 1.1 Problem Framing: Morning Briefing Refactor

## Project Goal
Refactor the legacy `morning_briefing.lobster` workflow into a deterministic, high-fidelity daily planning system that adheres to the **OpenClaw v2026.5.x** architectural standards.

## PEAS Framework

### Performance (P)
- **Primary Metric**: Time-to-Digest. The briefing should be generated in < 120 seconds.
- **Accuracy**: 100% adherence to existing calendar commitments (No double-booking).
- **Prioritization**: Correct identification of "Urgent/Important" tasks from Google Tasks.

### Environment (E)
- **Infrastructure**: Alienware Node (Pascal/Maxwell).
- **Services**: Google Calendar, Google Tasks, Weather API (BOM/OpenWeather), WhatsApp (Gateway).
- **Data**: Personal schedule, task backlog, and environmental conditions.

### Actuators (A)
- `gog` (Google Workspace CLI).
- `whatsapp-send-message` (Gateway).
- `lobster` (Workflow orchestration).

### Sensors (S)
- `gog` (List events/tasks).
- `weather-get-forecast`.
- `llm-classify-intent` (For parsing user approval/feedback).

## Success Criteria (Standardized Work)
1. **Deterministic Execution**: All steps use Lobster step handoffs (`$previous`).
2. **Zero Shell Scripts**: Direct tool invocation for all actuators.
3. **Hardware Pinning**: Summarization on Pascal, Extraction/Classification on Maxwell.
4. **Jidoka**: Every API call (Google/Weather) has a validation gate.
