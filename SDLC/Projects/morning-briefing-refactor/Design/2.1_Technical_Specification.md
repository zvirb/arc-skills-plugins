# 2.1 Technical Specification: Morning Briefing Refactor

## Architectural Topology
- **Host**: Alienware.
- **Workflow Engine**: Lobster 2026.5.x.
- **Actuators**: `gog` v0.13.x.
- **Specialization**:
    - **Orchestrator**: Pascal (GPU 0) - Qwen 35B.
    - **Specialists**: Maxwell (GPU 1) - Qwen 7B (INT4).

## Node Interface Specifications

### 1. `NODE_CAL_FETCH`
- **Command**: `gog calendar list primary --from today --to tomorrow --json --results-only`
- **Output Schema**: JSON array of Event objects.
- **Hardware**: CPU (Local Exec).

### 2. `NODE_TASK_FETCH`
- **Command**: `gog tasks list @default --status needsAction --json --results-only`
- **Output Schema**: JSON array of Task objects.
- **Hardware**: CPU (Local Exec).

### 3. `NODE_TIME_BLOCK` (The "Planner")
- **Prompt**: "Act as a professional executive assistant. Given the following calendar events and task list, propose a 1-day schedule. Group tasks into high-energy (morning) and low-energy (afternoon) blocks. Ensure 30-minute buffers between meetings."
- **Model**: `ollama/qwen2.5:35b` (Pascal).
- **Input**: `{"events": [...], "tasks": [...], "weather": "..."}`
- **Output Schema**: 
  ```json
  {
    "blocks": [
      { "title": "Focus: Task Name", "start": "ISO-8601", "end": "ISO-8601", "type": "task" }
    ],
    "conflicts": ["string"]
  }
  ```

### 4. `NODE_SUMMARIZE`
- **Prompt**: "Create a concise daily digest for WhatsApp. Use emojis. Highlight the first focus block and any urgent conflicts."
- **Model**: `ollama/qwen2.5:35b` (Pascal).

## Jidoka Validation Gates
1. **Gate: Context Validity**: If `NODE_CAL_FETCH` or `NODE_TASK_FETCH` returns an empty set, continue but insert "⚠️ No tasks/events found" into the Planner context.
2. **Gate: Schedule Integrity**: The Planner's output MUST be valid JSON and dates MUST be in the future. Validate via `llm-extract-json`.
3. **Gate: State Check**: Before `NODE_CAL_COMMIT`, re-run `NODE_CAL_FETCH` to ensure the proposed slots are still free.
