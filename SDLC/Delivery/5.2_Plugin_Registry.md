# 5.2 Plugin Registry & Compilation

## Objective
To manage the versioned catalog of native extensions and ensure they are optimized for the target runtime.

## Registry Precedence
OpenClaw resolves plugins in the following hierarchy:
1.  `plugins.load.paths` (Explicit overrides in `openclaw.json`).
2.  **Workspace Plugins**: `<workspace>/.openclaw/`.
3.  **Global Plugins**: `~/.openclaw/`.
4.  **Bundled Plugins**: `dist/extensions/`.

## Compilation Standards
- **Target**: Node.js 20+ (ESM).
- **Tooling**: `esbuild` or `tsc`.
- **Output**: Single-file bundles are preferred to reduce I/O latency on the Maxwell node.

## Manifest Validation
Every entry in the registry must contain an `openclaw.plugin.json` with:
- **`contracts`**: Declaring the exact tools provided.
- **`kind`**: Specifying the architectural slot (e.g., `context-engine`).
- **`dependencies`**: Listing any external binaries required (must match `SDLC 3.4`).

## Jidoka Check
- [ ] Run `openclaw plugins list --verbose` to ensure no manifest collisions.
- [ ] Verify that the `dist/` entry point is reachable by the gateway process.
