# 5.1 Automated Deployment Pipeline

## Objective
To establish a repeatable, low-friction deployment path that eliminates manual configuration errors (Standardized Work).

## The 4-Step Deployment Flow

### 1. Pre-Flight Check (Verification Gate)
- Ensure all Verification documents (SDLC Domain 4) are signed off.
- Run `openclaw skills check` one final time.

### 2. Compilation (Plugins Only)
- Execute `npm run build` to transform TypeScript to JavaScript.
- Verify the `dist/` directory contains the expected output.

### 3. Installation
- **MANDATORY**: Use the OpenClaw CLI for installation. 
- Command: `openclaw skills install <path>` or `openclaw plugins install <path>`.
- **Reason**: This automatically copies the files AND updates the `openclaw.json` configuration with their new relative location.

### 4. Agent Binding
- Manually (or via script) add the skill/plugin slug to the `agents.list[n].skills` array in `openclaw.json`.
- **Note**: Skills are NOT automatically inherited; explicit binding is required.

## Jidoka Validation
Deployment must fail if:
- The target directory is not writable.
- The `openclaw.json` update results in a malformed JSON file.
- The agent does not inherit the skill after a gateway reload.
