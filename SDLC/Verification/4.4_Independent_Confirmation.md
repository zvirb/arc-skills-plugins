# 4.4 Independent State Confirmation

## Objective
To retrieve physical proof that the agent's actions achieved the desired outcome.

## The Physical Verification Matrix

| Action Category | Verification Method | tool |
| :--- | :--- | :--- |
| **Filesystem Mutation** | Check for file existence and content checksum. | `fs-read`, `ls`. |
| **Google Workspace** | Navigate to the relevant URL and parse the UI. | `browser_subagent`. |
| **Database/API** | Query the endpoint for the specific record ID. | `curl`, `pg-query`. |
| **Code Implementation** | Run the code and verify it passes its own tests. | `npm test`, `pytest`. |

## The "Third-Party" Rule
Success is not confirmed by the agent that performed the task. It is only confirmed when a **separate, independent tool** returns data that matches the success criteria.

## Automated Verification Chain
1.  **Actuator Node** executes the task.
2.  **Monitor Node** waits for the environment to settle (1-2 seconds).
3.  **Auditor Node** invokes a verification tool.
4.  **Comparison Node** matches the results against the original Design Schema.
