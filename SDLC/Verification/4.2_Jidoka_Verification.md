# 4.2 Jidoka Loop Verification

## Objective
To ensure that the agentic system is resilient to failure and can self-heal without human intervention.

## Testing the "Andon" Loop
The "Andon" loop is the mechanism where an agent identifies a defect and stops to correct it. To verify this:

1.  **Inject Synthetic Errors**: Intentionally provide incorrect API keys, non-existent paths, or malformed JSON to a node.
2.  **Verify Detection**: Confirm the node captures the error and feeds it back to the LLM context.
3.  **Verify Correction**: Confirm the LLM generates a corrected instruction based on the error output.
4.  **Verify Termination**: Confirm the node exits gracefully if `max_retries` is reached, without hallucinating success.

## Failure Mode Analysis
Every node must be tested against the following failure modes:
- **Tool Timeout**: Does the node handle a hung subshell?
- **Empty Return Set**: Does the node distinguish between "No data found" and "Execution error"?
- **Schema Violation**: Does the node catch its own malformed output?
