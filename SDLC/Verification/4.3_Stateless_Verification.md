# 4.3 Stateless Verification Protocol

## Objective
To ensure that verification remains pure and does not pollute the production environment or the agent's long-term memory.

## Protocol Rules

### 1. The Isolation Rule
Verification must occur in a "Cold State":
- Clear the `Memory/core` before running E2E tests.
- Use temporary session IDs to prevent cross-session data bleeding.

### 2. The Observational Only Rule
**CRITICAL**: Verification tools must ONLY observe state. They are strictly forbidden from modifying or completing tasks. 
- **Bad Verification**: Using `browser` to click "Submit" on a failed form during a test.
- **Good Verification**: Using `browser` to read the "Submission Successful" message.

### 3. The Independent Auditor
Testing must be performed by an "Auditor" agent that did not participate in the original implementation. This prevents "Confirmation Bias" where the implementing agent assumes its own logic is correct.

## Execution
Use `wsl openclaw chat --session test-audit` to run isolated verification batches.
