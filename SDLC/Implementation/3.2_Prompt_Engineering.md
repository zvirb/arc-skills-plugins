# 3.2 Prompt Engineering

## Objective
To implement the specific reasoning logic required for autonomous execution on legacy hardware.

## Implementation Patterns

### 1. The Blindfold Execution Pattern
For worker nodes (Maxwell 12GB):
```markdown
# Role
System Logic Mapper (No Conversation)

# Instructions
1. Ingest the JSON payload from <input>.
2. Map the data to the <tool> arguments.
3. Execute and return only the raw JSON output.
```

### 2. The Structured Thinking Pattern
For the Orchestrator (Pascal 24GB):
```markdown
# Instructions
You must output all reasoning within a <thought> block. 
Your final response must be a valid JSON object matching the <output_schema>.
```

### 3. The "Repair" Prompt (Jidoka)
When a node fails:
```markdown
# Error Correction
The previous attempt failed with: <error>.
Reference the <schema> and <docs> to generate a corrected instruction.
Max retries: 3.
```

## VRAM Efficiency
- Keep system prompts concise to preserve the KV cache.
- Use **few-shot examples** sparingly, only for complex schema mappings.
- Prefer **structured descriptions** over long natural language paragraphs.
