# 3.4 Skill Authoring (Markdown)

## Objective
To teach the agent how to orchestrate existing tools using purely natural language instructions and YAML metadata.

## Skill Structure (`SKILL.md`)

### 1. Frontmatter (YAML)
- `name`: Human-readable name.
- `slug`: Unique identifier for the agent roster.
- `requires`: List of required binaries (e.g., `curl`, `jq`) and environment variables.
- `os`: Target OS (e.g., `windows`, `linux`).

### 2. Role & Context
Define the specific persona for this skill.

### 3. Instructions (SOP)
Provide the "Standardized Work" steps. Use numbered lists. 
- **Rule**: Be explicit about tool usage (e.g., "Use `gog tasks list` to verify state before creating a new task").

### 4. Verification
Include a section on how the agent should verify the success of the skill.

## Jidoka Validation
- [ ] Run `openclaw skills check <skill-name>` to verify syntax and dependency eligibility.
- [ ] Ensure no Python scripts are used to wrap inference.
- [ ] Verify that all required binaries are in the system path or `tools.exec.pathPrepend`.
