# 3.3 TypeScript Plugin Development

## Objective
To build native OpenClaw plugins for complex application state, native API execution, or state management.

## SDK Standards
- Use the `@openclaw/plugin-sdk`.
- Implement the `IPlugin` interface.
- Use `export default function register(api)` for tool registration.

## Implementation Rules
1.  **Strict Isolation**: Plugins must be written in TypeScript (ESM) and reside in the `Plugins/` directory.
2.  **Manifest Contracts**: Every plugin must include a `contracts` block in `openclaw.plugin.json` declaring its tools.
3.  **No Zombie Subshells**: Use the `api.runtime.taskFlow` interface for long-running operations to ensure state persistence and cleanup on exit.
4.  **ZSP (Zero Standing Privileges)**: Inject secrets from `Secrets/` at request-time; never store credentials in the plugin code.

## Compilation Requirement
Plugins must be compiled to JavaScript (`npm run build`) before installation. Ensure the entry point in the manifest maps to the `dist/` directory.

## Jidoka Check
- [ ] Does the plugin survive the 8-step load pipeline? (`openclaw plugins list --verbose`)
- [ ] Does it handle connection timeouts gracefully?
- [ ] Is it written in ESM format?
