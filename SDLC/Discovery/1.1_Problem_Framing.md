# 1.1 Problem Framing

## Objective
To clearly define the boundaries of the problem and the operational scope of the autonomous agent.

## The PEAS Framework
Every agent project must start by defining its PEAS components:

| Component | Description | Example (PDF Parser Agent) |
| :--- | :--- | :--- |
| **Performance** | The metric for success. | Accuracy of JSON extraction, processing speed. |
| **Environment** | Where the agent operates. | Local filesystem, Maxwell 12GB GPU node. |
| **Actuators** | Tools the agent uses to affect the environment. | `deepseek-ocr`, `fs-write`, `json-validate`. |
| **Sensors** | How the agent perceives the environment. | `fs-read`, `tool-output-stream`, `logs`. |

## Goal Atomization
Avoid general-purpose goals. 
- **Bad Goal**: "Build a research assistant."
- **Good Goal**: "Extract chemical structures from a directory of PDF patents and output a SMILES-compliant CSV."

## Standardized Work Checkpoint
- [ ] Is the problem frameable as a single-objective task?
- [ ] Are the performance metrics deterministic?
- [ ] Is the environment clearly bounded (e.g., local vs remote)?
