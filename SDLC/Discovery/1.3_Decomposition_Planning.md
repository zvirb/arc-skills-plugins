# 1.3 Decomposition Planning

## Objective
To break down complex objectives into the smallest possible, testable atomic operations (Standardized Work) within a deterministic execution graph.

## The Atomic Node Pattern
A "Node" is the fundamental unit of work in OpenClaw. It must follow the **Single Responsibility Principle**:
- A node either **formats data**, **executes a tool**, or **evaluates a result**.
- **NEVER** combine these operations into a single node.

## Decomposition Strategy
1.  **Identify the Critical Path**: List the absolute minimum steps to reach the goal.
2.  **Atomize Nodes**: For each step, determine if it can be broken down further.
3.  **Define Orchestration**: Use **.lobster** macros to chain these nodes deterministically.
4.  **Define Hand-offs**: Specify exactly how data is piped (e.g., using **PluginArtifact** schema or the **lobster://** protocol for large payloads).

## The Specialist Roster Architecture
For complex workflows, use a decoupled specialist roster:
- **Orchestrator**: Managed via `.lobster`, coordinates the flow.
- **Workers (Specialists)**: Highly scoped agents with strictly distinct sessions.

## Verification Protocol
- [ ] Does every node have exactly one output schema?
- [ ] Can every node be unit-tested in isolation on the **Alienware** node?
- [ ] Is the total context required for any worker node capped at **4,000 tokens** (for VRAM stability)?
- [ ] Are all inter-agent communications handled via deterministic Lobster piping?
