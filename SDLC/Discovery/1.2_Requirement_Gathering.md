# 1.2 Requirement Gathering

## Objective
To move from informal intent to structured artifacts that serve as technical contracts for agentic execution.

## Artifact-Driven Discovery
The discovery process must produce the following standardized artifacts:

### 1. Spec First Contract
Before code is written, a technical specification must be authored.
- **Input Schema**: Strict definition of expected input (JSON/YAML).
- **Output Schema**: Strict definition of expected result (JSON/YAML).
- **Transformation Logic**: Mathematical or logical description of the transition from Input to Output.

### 2. Personas & Policies
- **`PERSONAS.md`**: Defining the specific roles involved (e.g., "The Technical Lead", "The Security Auditor").
- **`POLICIES.md`**: Defining the constraints (e.g., "All code must be TypeScript", "No external API calls allowed").

## Jidoka Validation
Requirements are not accepted unless they pass the **"Blindfold Test"**:
> If the agent was blindfolded and only given the tools and these requirements, could it reach the success state without asking for clarification?

## Verification Steps
1.  **Schema Review**: Ensure all I/O schemas are compliant with `openclaw.json`.
2.  **Constraint Audit**: Ensure all policies are enforceable by the current `Agents.md` runtime.
